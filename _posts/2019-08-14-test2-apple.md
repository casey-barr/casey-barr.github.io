### Introduction 

As my first *official* blog post, I thought that I'd take a look and explore some data personal to me; my own Apple Health data, collected primarily from my two most frequently used Apple Devices, my iPhone and my Apple Watch.

To start, there'll be a brief write-up on how to get the data collected from Apple's Heatlh app, and then I'll move on to doing some EDA on that data in a Jupter notebook, as well as a machine learning model that I wrote in TensorfFlow to see if my habits can be predicted. 

### Data Collection

I will be using [this guide](http://www.markwk.com/data-analysis-for-apple-health.html), written by [Mark Koester](http://www.markwk.com/), to extract and format the raw data stored on my Apple Devices.

We can begin with opening the iPhone Health app, selecting the User Icon in the top right corner of the screen, and then selecting the 'Export Health Data' option at the bottom of the page.

![health_app](health_app.png)

The Health app then collects all of the data stored on your device, and will provide you with a file named `export.zip` after a few moments of processing. You can then choose to share it with yourself through any of Apple's filesharing methods (email, Airdrop, etc.). I chose to email my data to myself, and downloaded the zip to my local computer.

### Data Processing and Formatting

As mentioned above, we will be using a script to parse and process the raw data from the Health app.

Navigate your browser to [https://github.com/markwk/qs_ledger/tree/master/apple_health](https://github.com/markwk/qs_ledger/tree/master/apple_health) and download the all of the folders and code from that sub-repository. Alternatively, you could clone the subdirectory directly from the command line with:

`git clone https://github.com/markwk/qs_ledger apple_health/`

Now we can unzip our previously exported Health data (the 'export.zip' file we got from the Health app), and copy the the folder `apple_health_export` into the `apple_health` repository we just downloaded. We will also need to copy the `export.xml` file into the `data` folder in our directory.

![apple_health_folder](apple_health_folder.png)

To begin the actual processing of the data, we wil use a Jupyter notebook (I'm actually using the Jupyter lab in these screenshots, but the same process applies). If you have Anaconda installed, open your terminal and type:

`jupyter notebook`

If you don't have Anaconda installed, you can download it here: [https://www.anaconda.com/](https://www.anaconda.com/).

Now, in your browser, locate the `apple_health` folder and open the notebook called `apple_health_extractor.ipy`.

![Notebook](jupyter_notebook_extractor.png)

Select the cell with the code: `%run -i "apple-health-data-parser" "data/export.xml"` and then press CTRL+ENTER to run the code. This code will extract and export CSV files for each health metric that was included in the `export.xml` file we generated from the Health app. The process should take a couple of minutes, depending on the size of the data that was exported.

To begin our EDA, we can import the "standard" Python packages numpy and pandas, along with glob and pytz. 

Pytz is a Python package that allows for timezone transformations. Our Health timezone data is formatted according to UTC, so we will write some simple lambda functions that allow us to convert them into the timezone of our choice (for me it's America/New York).
